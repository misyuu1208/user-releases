Ilmu API Platform Release Notes

May 2026

Billing, compliance & developer experience updates

This release improves billing reliability, adds LHDN e-invoice compliance, tightens subscription lifecycle management, and improves the developer onboarding experience.

Billing & Payments

New

Invoices and receipts now include official invoice and receipt numbers, are available as downloadable PDFs, and support the LHDN e-invoice workflow for Malaysian tax compliance.

New

Subscriptions are now automatically suspended on the first failed recurring charge, giving you a clear signal to update your payment method before access is affected.

New

Cancelled subscriptions now expire at the correct period-end date rather than remaining active indefinitely.

Fix

Receipt PDF timestamps now display in Asia/Kuala Lumpur time instead of UTC.

Fix

Razorpay webhooks arriving outside the 5-minute validity window are now correctly rejected, improving payment security.

Fix

Resolved a bug that was doubling token balances for users with stacked subscriptions.

Fix

Add-on packs purchased under a past_due subscription are now correctly nested rather than appearing as orphaned line items.

Fix

One-off pack models are now correctly surfaced on /subscriptions/current and /v1/models, so clients always see the full set of models available to them.

Plans & Provisioning

New

Free plan provisioning is now gated — users claiming an invite code whose phone number is already associated with an existing account will receive a clear notification instead of silently failing, covering both standard and email-last verification flows.

Improved

One-off token allocations have been refactored for consistency, and lifetime pack usage math has been corrected. Inactive entitlements are now surfaced correctly in account views.

Developer Experience

New

The dashboard now shows a tool-aware Quick Start card that adapts its guidance based on which tools and API keys you have configured, making it faster to get to your first successful call.

Improved

Streaming (SSE) responses are now more robust — partial lines are buffered correctly in the search interceptor, preventing truncated events from reaching clients.

Documentation & Console

Improved

Billing page FAQ updated with clearer information. The plan comparison modal and "What you'll get" link have been removed to simplify the upgrade flow.

Improved

Coding plan documentation and terminology have been refreshed to reflect current naming and capabilities.
